## Key Notes

> **Render DB** — Currently running on the `mvp_manager` Render PostgreSQL instance (a pre-existing older instance) because freshly provisioned Render PG instances silently drop connections for several hours after creation. No user impact. Migrate to a dedicated `onsiteit-db` instance once Render resolves the infrastructure bug.

> **No test suite** — The project has zero automated tests. This is acceptable for MVP but should be addressed before 1.0 or before adding email/payment integrations where regressions would be costly.
